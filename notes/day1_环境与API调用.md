## 二、核心概念（费曼输出区）

### 概念 1：Python 虚拟环境（venv）

- **是什么**：
  虚拟环境是 Python 提供的一种"项目级依赖隔离机制"。
  本质是在项目目录下创建一个独立的 Python 解释器副本和独立的包仓库，
  让这个项目使用的包不影响系统全局，也不影响其他项目。

- **为什么需要它**：
  不同的 Python 项目可能依赖同一个包的不同版本。
  比如项目 A 需要 langchain 0.1.0，项目 B 需要 langchain 0.3.0。
  如果都装在系统全局 Python 里，后装的会覆盖先装的，
  导致两个项目无法同时正常运行。虚拟环境给每个项目一个独立"小盒子"，
  彻底解决依赖冲突问题。

- **怎么用**：
  1. 创建：`python -m venv venv`（用 venv 模块在当前目录创建名为 venv 的文件夹）
  2. 激活（Windows PowerShell）：`.\venv\Scripts\Activate.ps1`
  3. 验证：`where.exe python`，第一行是 venv 路径即成功
  4. 退出：`deactivate`

- **隔离的底层原理**（亮点 ⭐）：
  激活虚拟环境的本质是修改 PATH 环境变量，
  把 venv\Scripts 目录"插队"到 PATH 最前面。
  这样在命令行输入 python 时，系统优先找到 venv 里的解释器，
  而不是系统的，从而实现"逻辑隔离"。

- **类比**：
  虚拟环境像每个项目独立的"工具箱"。
  想象你是个装修工，有两个客户：
  - 客户 A 家是欧式风，需要"老款锤子"
  - 客户 B 家是现代风，需要"新款锤子"
  如果你只用一个工具箱，两把锤子放进去会互相挤占空间，甚至混淆。
  虚拟环境就是给每个客户单独配一个工具箱，
  进入客户 A 家就拿 A 的工具箱，进入 B 家就拿 B 的工具箱。
  工具箱之间互不干扰。
